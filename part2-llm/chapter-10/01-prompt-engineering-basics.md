# 10.1 提示工程基础

> "提示工程是与AI对话的艺术——同样的模型，不同的提示，截然不同的结果。"

## 什么是提示工程

### 定义

```
提示工程（Prompt Engineering）：

设计和优化输入提示（prompt），
以引导大语言模型产生期望输出的技术和方法。

为什么重要？
- 模型能力是固定的
- 提示质量决定输出质量
- 好的提示 = 好的结果
```

### 提示的组成

```
一个完整的提示通常包含：

1. 指令（Instruction）
   "翻译以下文本到中文"

2. 上下文（Context）
   "你是一位专业的技术翻译"

3. 输入数据（Input Data）
   "Hello, world!"

4. 输出格式（Output Format）
   "只返回翻译结果，不要解释"

示例：
你是一位专业的技术翻译（上下文）。
请将以下英文翻译成中文（指令），
只返回翻译结果（格式）。

文本：Hello, world!（输入）
```

## 基础技巧

### 清晰具体

```
❌ 差的提示：
"写一些关于Java的东西"

问题：
- 太模糊
- 不知道写什么
- 不知道写多长

✅ 好的提示：
"请用200字介绍Java的Stream API，
包括它的主要用途和一个简单示例。"

优点：
- 具体主题
- 明确长度
- 包含要求的内容
```

### 提供示例

```
Few-shot提示：

请按照以下格式将自然语言转换为SQL：

示例1：
问题：查询所有年龄大于30岁的用户
SQL：SELECT * FROM users WHERE age > 30;

示例2：
问题：统计每个部门的员工数量
SQL：SELECT department, COUNT(*) FROM employees GROUP BY department;

现在请转换：
问题：查找工资最高的前10名员工
SQL：
```

### 角色设定

```java
/**
 * 角色设定示例
 */
public class RolePrompting {
    
    /**
     * 不同角色，不同输出
     */
    public void demonstrateRoles() {
        String question = "解释什么是多线程";
        
        // 角色1：大学教授
        String professorPrompt = """
            你是一位计算机科学教授，
            用学术语言详细解释概念，
            包括理论背景和应用场景。
            
            请解释：""" + question;
        
        // 角色2：初中老师
        String teacherPrompt = """
            你是一位初中信息技术老师，
            用简单易懂的语言解释，
            使用生活中的比喻。
            
            请解释：""" + question;
        
        // 角色3：资深工程师
        String engineerPrompt = """
            你是一位有10年经验的Java工程师，
            重点讲解实际开发中的应用和最佳实践，
            包含代码示例。
            
            请解释：""" + question;
        
        // 同一问题，不同角色会得到不同风格的回答
    }
}
```

## 提示设计模式

### 1. 零样本（Zero-shot）

```
直接提问，不给示例：

"将以下文本分类为正面、负面或中性：
这家餐厅的服务太棒了！"

适用场景：
- 模型已经理解的任务
- 简单直接的分类
- 通用知识问答
```

### 2. 少样本（Few-shot）

```
提供几个示例：

情感分析示例：

文本：这部电影真好看
情感：正面

文本：等了一个小时还没上菜
情感：负面

文本：今天天气不错
情感：中性

文本：这个产品质量一般
情感：

适用场景：
- 特定格式的任务
- 需要学习模式的任务
- 提高准确率
```

### 3. 思维链（Chain-of-Thought）

```
引导模型逐步思考：

"问题：一个农场有鸡和兔，头共35个，脚共94只。
鸡和兔各有多少只？

请逐步思考：
1. 设鸡有x只，兔有y只
2. 根据头的数量：x + y = 35
3. 根据脚的数量：2x + 4y = 94
4. ..."

适用场景：
- 数学问题
- 逻辑推理
- 复杂决策
```

## 常见错误

### 错误1：提示过于复杂

```
❌ 错误：
"请写一篇关于Java的文章，要包括历史、
特性、应用场景、与其他语言的对比、
未来发展趋势，还要适合初学者阅读，
字数在1000-1500字之间，
使用Markdown格式，
包含至少3个代码示例..."

问题：
- 要求太多，模型可能遗漏
- 难以控制输出质量

✅ 改进：
分步骤提示：
第1步："请列出Java的5个主要特性"
第2步："基于这些特性，写一个100字的介绍"
第3步："为每个特性提供一个代码示例"
```

### 错误2：缺乏具体性

```
❌ 错误：
"写一些代码"

✅ 改进：
"请用Java编写一个单例模式的实现，
要求线程安全且延迟加载，
包含详细的注释说明"
```

### 错误3：忽略边界情况

```
❌ 错误：
"解释如何处理异常"

✅ 改进：
"请解释Java中Checked Exception和
Unchecked Exception的区别，
并给出每种情况下的最佳处理实践，
包括什么时候应该捕获、什么时候应该抛出"
```

## 评估提示质量

### 检查清单

```
好的提示应该：

□ 明确具体
  - 清楚说明要做什么
  - 指定输出格式

□ 提供上下文
  - 设定角色
  - 说明背景

□ 包含示例（如需要）
  - Few-shot示例
  - 输入输出格式

□ 设置约束
  - 长度限制
  - 风格要求
  - 禁止事项

□ 可验证
  - 输出可以检查对错
  - 有明确的评估标准
```

## 小结

本章我们学习了：

1. **提示工程定义**：设计和优化输入提示
2. **提示组成**：指令、上下文、输入、格式
3. **基础技巧**：清晰具体、提供示例、角色设定
4. **设计模式**：零样本、少样本、思维链
5. **常见错误**：过于复杂、缺乏具体性

**关键认识：**
提示工程是与大模型交互的核心技能，好的提示能显著提升输出质量。

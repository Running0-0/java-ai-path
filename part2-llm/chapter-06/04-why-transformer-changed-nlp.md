# 6.4 设计思考：为什么Transformer改变了NLP

> "Transformer不仅是一种架构，更是NLP范式的转变——从顺序处理到并行理解，从局部关注到全局感知。"

## Transformer的革命性

### 对比传统方法

```
Transformer之前的NLP：

RNN/LSTM时代：
- 顺序处理：一个词一个词地读
- 长距离依赖困难
- 难以并行计算
- 训练速度慢

Transformer之后：
- 并行处理：一次性看所有词
- 全局注意力：任意两个词直接关联
- 完全并行：GPU加速效果显著
- 训练速度快10倍以上
```

### 关键突破

| 方面 | 传统方法 | Transformer |
|------|----------|-------------|
| 处理方式 | 顺序 | 并行 |
| 依赖距离 | 受限 | 全局 |
| 训练速度 | 慢 | 快 |
| 可扩展性 | 差 | 好 |
| 预训练效果 | 有限 | 卓越 |

## 架构创新分析

### 自注意力的威力

```
传统注意力：
查询 → 编码器输出 → 注意力权重 → 上下文

自注意力：
每个词都可以是查询、键、值
词与词之间直接计算关系

优势：
1. 无需递归，完全并行
2. 任意距离依赖，路径长度为1
3. 可解释性强，注意力权重可视化
```

### 位置编码的巧妙

```
RNN：位置信息天然包含在顺序中
Transformer：需要显式注入位置信息

解决方案：正弦位置编码
- 每个位置有唯一编码
- 可以外推到训练时未见过的长度
- 相对位置信息可学习
```

## 工程实践影响

### 预训练范式的兴起

```
Transformer之前：
- 针对特定任务训练模型
- 数据需求大，效果有限
- 迁移困难

Transformer之后：
- 先在大规模语料预训练
- 再针对任务微调
- 小数据也能有好效果
```

### 规模效应

```
Transformer展现了惊人的规模效应：

参数量增长 → 能力跃升
- 1亿参数：基础语言理解
- 10亿参数：流畅文本生成
- 100亿参数：推理能力显现
- 1000亿参数：涌现多种能力

这就是大语言模型的由来
```
